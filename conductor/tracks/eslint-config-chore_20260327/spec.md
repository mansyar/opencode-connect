# Track: eslint-config-chore_20260327

## Overview

This track fixes the broken ESLint configuration and sets up comprehensive pre-commit checks to enforce code quality.

## Root Cause

ESLint v10.1.0 is installed but `eslint-config-expo` v55.0.0 only supports ESLint v9.x. The flat config API changed in ESLint 10, causing `expo is not a function` error.

## Functional Requirements

### 1. Fix ESLint Configuration

- **Downgrade ESLint** from v10.1.0 to v9.x to ensure compatibility with `eslint-config-expo` v55
- Update `eslint.config.mjs` if needed after ESLint downgrade
- Verify ESLint runs without errors: `pnpm lint`

### 2. Add Prettier Configuration

- Create `.prettierrc` with Expo-recommended settings
- Verify Prettier works: `pnpm format`

### 3. Configure lint-staged (Pre-commit Hooks)

Install and configure `lint-staged` to run on staged files:

- **ESLint + Prettier**: `pnpm lint --fix && pnpm format`
- **TypeScript type check**: `pnpm typecheck` (via `tsc --noEmit`)
- **Line count check**: Custom script rejecting files > 500 lines

### 4. Add typecheck Script

Add to `package.json` scripts:

```json
"typecheck": "tsc --noEmit"
```

### 5. Create Line Count Check Script

- Create `scripts/check-line-count.js`
- Script scans staged files for `.ts`, `.tsx`, `.js`, `.jsx`
- Fails if any file exceeds 500 lines

## Files to Create/Modify

| File                          | Action                                                        |
| ----------------------------- | ------------------------------------------------------------- |
| `package.json`                | Downgrade ESLint to v9, add lint-staged, add typecheck script |
| `eslint.config.mjs`           | Update if needed for ESLint 9 compatibility                   |
| `.prettierrc`                 | Create with Expo defaults                                     |
| `scripts/check-line-count.js` | Create for line count validation                              |
| `.git/hooks/pre-commit`       | Auto-generated by lint-staged                                 |

## Out of Scope

- Modifying application source code to pass linting
- Adding/modifying ESLint rules beyond defaults

## Acceptance Criteria

- [ ] `pnpm lint` runs without errors
- [ ] `pnpm typecheck` runs without errors
- [ ] `pnpm format` formats code correctly
- [ ] Pre-commit hook runs all 3 checks (lint, typecheck, line count)
- [ ] Files > 500 lines are rejected on commit
